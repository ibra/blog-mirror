!brahimDev 09-Dec-20 03:27 PM
So yesterday I thought that i should probably migrate YourShot to Unity's (kinda) New Input System which isnt in preview anymore so it should be a bit more stable, and obviously would require a refactor of code, but I had m o t i v a t i o n, so like why not. So i made a new branch and shit and i set out migrating YS to the new system. My initial thoughts were that this was very complicated relative to the old system, however that system was really hard to work with for multiple inputs, and ESPECIALLY if you want to have any sort of rebinds for your game, and overall after i while i just preferred this new Event callback thingy based Input System more in the long run. (edited)
There are definitely some uhh weird ass bugs like Unitys Debugging tool still thinking that im using the Old Input System, and a few other odd things. Thankfully unity allows us to use both Input Systems at once so if there are any unity problems / package problems relating to the new system you're running both Input Systems, which isnt exactly ideal, but like eh its good for now. Ive already done basic movement / interaction / Aiming / Equipping logic, just need to add Menu and Combat logic with the new Input System. Should be able to fully migrate in like 3 days or so. Also, for those of you that hated the random joystick button shit in the Old Input System (having to look up joystick numbers for each button on a controller, and those being wrong half the time), this new system has a "Listen" feature which basically just does what it says and listens for any Input no matter the input device you use (Seems to me that it supports quite a few devices).

!brahimDev 17-Jan-21 07:46 PM
The time has come to add a proper method of saving to YourShot. Previously i had been using the Pause Menu to allow for saving, but I feel like it just doesnt fit the game, and also breaks immersion for a while when you’re opening a menu, completely freezing the game, and then pressing save. I looked into a bunch of approaches. This video in particular was especially helpful: https://www.youtube.com/watch?v=PJZByWHtzb8 From what I can tell these are most major ways to handle saving in your game Autosaving Manual Saving Anytime, Anywhere Autosaving + Manual Saving Auto Manual Saving Prompts Quicksaves Save Rooms Save Points And the one that really caught my eye were save points. Something about the feeling when you have to face off multiple encounters without saving when the next save point is to the end of the level/room is just something that I felt like would make for an enjoyable experience. But like some other saving options, this one can be very annoying if not done right. I recalled that I had recently implemented a choice system for YourShot already, so why not use that, and enable you to interact with a Save Point that shows information on your current whereabouts and whatnot, and ofcourse allows you to save the game. So, lets get right to it! First, I needed a Sprite for the save point, so I made this weirdish sprite that gets darker each frame, before looping back. It actually turned out alright in game though.
Next, I used the choice system that i had recently made for setting up a dialogue for the save point interaction. Heres how it looks in game:
Then I just needed to do some backend stuff like hooking the savepoints OnDialogueEnd event to a GameManager.Save() function. And yeah that was about it. I turned this save point into a Unity Prefab and now I can populate my levels with save points. I want to keep a healthy balance of save points for YourShot. I dont want them to be too scarce, but I also want to make it a challenge to reach them. For frequency Im thinking of something from an Undertale Savepoint to a Dark Souls Bonfire. That was about it for now, I really want to get to work now on implementing a unique health system as well as some fun combat, and that's probably what im gonna work on next. I currently have exams that’ll probably last around like 2 weeks, so I wont really get much time to work on the game at all, but I try to jot down any ideas i have, and they’re pretty pog ngl .
